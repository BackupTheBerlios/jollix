  Installing jollix  
     
 
Have you ever wished there was a distro with Gentoo's Portage System and automatic hardware detection, and was easier to install than Gentoo? Guess what. Your wish came true.
A German open source team kicked ass with the creation of a gaming and multimedia distro. jollix boots up and shuts down Gentoo kernel 2.6.5 with an animated graphical splashscreen, and automatically logs in KDE 3.2.2 with Plastik 0.3.9 theme in X.org 6.7.0. Window drop-shadow is on by default. In the bottom-left of the desktop is a CPU, memory and network monitor, a SuperKaramba 0.33 theme. In the upper-right is a set of system setup tools.
KDE has all the basic programs the average user will ever need from a desktop PC. From Web browsers, e-mail/IM/IRC clients, music and movie players, TV and video-conferencing programs to productivity suite. Some are one click away from the bottom toolbar. As you can see, jollix makes a great distro for hobbyists, students and parents.
So many distros throw KDE in without the slightest customization, jollix doesn't fall flat on big screen. Most programs are set up. Playing DVD or VCD movies is just one click in Xine 0.9.23. There are also i686 optimization, XQF 0.9.14 Counter Strike / Half Life / Quake / frag-'em-some-more front-end to QStat 2.5c, KBear 2.1.1 ftp client, kopete 0.8.2 instant messaging, LinNeighborhood 0.6.5, GIMP 2.0.0, Quanta Plus 3.1.4 Webpage builder, K3b 0.11.9 CD/DVD recording tool, CUPS 1.1.20 printer support, Samba 3.0.2a, SANE 1.0.13 scanner support, XMMS 1.2.8 audio player and much more.
jollix can be run as either a live CD system or manually installed as a permanent system. Are you game?

Getting jollix
Get jollix ISO and md5sum from download mirrors, and verify the md5sum:

md5sum -c jollix-0.3.beta2.i686.md5

If it returns ok or nothing, burn the ISO on CDR/W. If it reads "fail", get the ISO and md5 from another mirror.

Going Live
Boot the Live CD, and at boot prompt enter this to start jollix in English:

jollix lang=us

Downloading Kernel, Portage and Doc Tarballs
In advance of installing jollix on hard drive, download kernel-spare-03beta2.tar.bz2, portage-spare-03beta2.tar.bz2, and doc-spare-03beta2.tar.bz2.

Partitioning Hard Disk
jollix install requires a pair of Linux and swap partitions. Click Konsole/Terminal button in the bottom toolbar, become root with password "jollix", set the current date and time in this format MMDDhhmmYYYY (month, day, 24-hour, minute, year), and run cfdisk to create a Linux partition and a swap partition, replacing hdx with hda for the first hard drive or hdb for the second:

su -
cd /mnt
date 123123592004
cfdisk /dev/hdx

Choose a 5 GB or larger partition, [Type], type "83", [Write], enter "yes", choose a small swap partition, [Type], type "82", [Write], enter "yes", [Quit].
Make the swap partition, turn it on ignoring busy device warning, format the Linux partition as ext3, make jollix directory in /mnt, mount it, and make jollix/boot directory, replacing hdx?'s with your swap and Linux partitions:

mkswap /dev/hdx?
swapon /dev/hdx?
mke2fs -j /dev/hdx?
mkdir jollix
mount /dev/hdx? jollix
mkdir jollix/boot

Copying Files and Chrooting to jollix
Go to the Compressed LOOPback filesystem, recursively copy system files while preserving file permissions and ownerships, mount the process information pseudo filesystem (proc), copy resolver config file, change root directory to jollix, and update the environment settings:

cd cloop
cp -dvR * /mnt/jollix/ --preserve=mode,ownership
cd /mnt/jollix
mount -t proc proc proc
cp /etc/resolv.conf etc
chroot /mnt/jollix /bin/bash -l
env-update

Installing GRand Unified Bootloader
GRUB is the default boot loader for jollix. If the jollix partition is /dev/hda5, x,x will be 0,4. Install GRUB, replacing the x's with jollix partition:

grub
root (hdx,x)
setup (hdx,x)
quit

Edit grub.conf, replacing the x's and /dev/hdx? with the jollix partition without breaking the kernel line:

nano -w /boot/grub/grub.conf

default 0
timeout 0
splashimage=(hdx,x)/boot/grub/splash.xpm.gz
title=jollix
root (hdx,x)
kernel (hdx,x)/boot/vmlinuz-2.6.5-r1 root=/dev/hdx? splash=silent
 video=vesa:ywrap,mtrr vga=0x317
initrd=(hdx,x)/boot/initrd-1024x768

Configuring Filesystem Table
Filesystem Table determines what partitions Linux can access. Edit fstab, replacing /dev/hdx?'s with the jollix and swap partitions, assuming there is only one installed optical drive:

nano -w /etc/fstab

#/dev/hda1          /boot       ext2    noauto,noatime        1 2
/dev/hdx?           /           ext3    noatime               0 1
/dev/hdx?           none        swap    sw                    0 0
/dev/cdroms/cdrom0  /mnt/cdrom0 iso9660 noauto,ro,user,unhide 0 0
#/dev/cdroms/cdrom1 /mnt/cdrom1 iso9660 noauto,ro,user,unhide 0 0
/dev/fd0            /mnt/floppy vfat    noauto,rw,user,unhide 0 0
none                /sys        sysfs   defaults              0 0
proc                /proc       proc    defaults              0 0
tmpfs               /dev/shm    tmpfs   defaults              0 0

Copying Init Scripts, Making /var and Rebooting
Copy local, modules and checkroot init scripts to /etc/init.d, create var directory and its variably-sized contents (mail, logs, databases, etc.), exit out of chroot, unmount the filesystems, and exit out of root:

cd /spare/etc/
cp local-org /etc/init.d/local
cp modules-org /etc/init.d/modules
cp checkroot-org /etc/init.d/checkroot
cd /var
tar -xjpf /spare/var-spare.tar.bz2
exit
cd /
umount /mnt/jollix/proc /mnt/jollix
exit

Reboot computer by clicking "K" button in bottom toolbar > Logout "user"... > Restart Computer > [OK]. Remove the CD.

Decompressing Kernel, Portage and Doc Tarballs
jollix can mount all Windows partitions. If you downloaded the three files in Windows, mount the Windows partition by clicking "Devices/Laufwerke" icon on desktop, and replace /home/user/downloads/ with the right path.
Open Konsole, become root with password "jollix", change root password, copy the three archives while not breaking the cp line, and decompress kernel source code, the Portage System and documentation in their respective directories:

su -
passwd
cd /home/user/downloads/
cp kernel-spare-03beta2.tar.bz2 portage-spare-03beta2.tar.bz2
 doc-spare-03beta2.tar.bz2 /spare
cd /spare
tar -xjpf kernel-spare-03beta2.tar.bz2 -C /usr/src
tar -xjpf portage-spare-03beta2.tar.bz2 -C /usr
tar -xjpf doc-spare-03beta2.tar.bz2 -C /usr/share

Configuring Network and Synching Portage Tree
Nice job. You just installed a jollix system. If your box isn't on a network or Internet-ready, skip to the next part.
Edit the net file with the IP, broadcast, netmask and gateway of your ISP, cable modem or router:

nano /etc/conf.d/net

iface_eth0="191.168.0.100 broadcast 192.168.0.255 netmask
 255.255.255.0"
gateway="eth0/192.168.0.1"

Edit the resolv.conf file with your ISP's two sets of DNS:

nano /etc/resolv.conf

nameserver 192.168.0.253
nameserver 192.168.0.254

Make network start up at boot-up, start it up, and, if you're on broadband, synchronize the Portage Tree for software management:

rc-update add net.eth0 default
/etc/init.d/net.eth0 start
emerge sync

Installing Common UNIX Print System
If you have a printer connected to the your rig, make CUPS auto-start on next reboot, and run CUPS daemon:

rc-update add cupsd default
/etc/init.d/cupsd start

At the boot-up and shut-down splashscreen, you can push F2 key to see the verbose mode of boot-up or shut-down messages.

Configuring Keyboard Mapping, Console Font and Clock
If you ever type in full-screen command line interface, you may find your keyboard uncooperative. The default keyboard mapping is German. Click the Konsole/Terminal button in the bottom toolbar, become root, find your keyboard mapping, and put it in KEYMAP parameter, uncomment default console font, and set CLOCK to local in the global start-up script:

su
ls /usr/share/keymaps/i386
nano -w /etc/rc.conf

KEYMAP="us"
CONSOLEFONT="default8x16"
CLOCK="local"

Searching Files with slocate
Secure locate indexes files and quickly searches for files. Create the slocate database as root:

slocate -u -e /mnt &

Run the above line after software management. Search for a file quickly:

slocate filename

Reconfiguring X in jollix 0.3.x and Later
If for whatever reason X Window is messed up, reconfigure X automatically:

mkxorgconf.sh

Setting Up X Keyboard Layout, Mouse, Screen and Color Depth
Initially, the X keyboard layout is German, any mouse wheel is disabled, mouse speed is slow, screen is off center, and default color depth is 16 bit. My mouse is a Microsoft Wheel Mouse Optical on PS/2.
Run video mode tuner, center the screen, [Apply], [Show], [Quit], highlight to copy your mode line, edit ZAxisMapping and Resolution in InputDevice section, click middle mouse button to paste your ModeLine in Monitor section, and change DefaultColorDepth to 24 in Screen section, of XF86Config:

xvidtune
nano -w /etc/X11/XF86Config

Option "XkbLayout" "us"
...
Section "InputDevice"
    Identifier "PS/2 Mouse"
    Driver     "mouse"
    Option     "Protocol"        "IMPS/2"
    Option     "ZAxisMapping"    "4 5"
    Option     "Resolution"      "256"
    Option     "Device"          "/dev/psaux"
    Option     "Emulate3Buttons" "true"
    Option     "Emulate3Timeout" "70"
    Option     "SendCoreEvents"  "true"
EndSection
...
    ModeLine "1024x768" 94.50 1024 1088 1184 1376 768 769 772 808
 +hsync +vsync
...
DefaultColorDepth 24

Click "My Computer" > click home button in the top toolbar, drag the "file:/home/user" folder icon from Location toolbar and drop it in empty desktop > click "Link Here". Right-click the new "user" folder icon on desktop > Properties > type "My Documents" in the text box of General tab, [OK]

Enabling CD/DVD Burner in jollix 0.2.x
Do NOT do it in jollix 0.3.x and later since kernel 2.6.x does NOT need ide-scsi module. In order for K3b to burn optical disks, you enable SCSI emulation in GRUB. Add hdc=ide-scsi to root parameter of grub.conf in Konsole:

nano -w /boot/grub/grub.conf

default 0
timeout 10
splashimage=(hdx,x)/boot/grub/splash.xpm.gz
title="Start jollix"
root (hdx,x)
kernel (hdx,x)/boot/vmlinuz-2.4.20-gaming-r3 root=/dev/hdx?
 hdc=ide-scsi splash=silent video=vesa:ywrap,mtrr vga=0x317
initrd=/boot/initrd-1024x768

And load the ide-scsi module at boot-up by uncommenting ide-scsi in modules.autoload:

nano -w /etc/modules.autoload

ide-scsi

Reboot the jollix system:

reboot

Setting Up K3b in jollix 0.2.x
In jollix 0.3.x and later, K3b is all set to go. Now that SCSI emulation is enabled, and ide-scsi module is loaded, you have to give permission to users for burning optical disks and set up K3b. Click "K" button > Applications > Multimedia > K3b > check "Do not ask again", [Yes], type root password, [OK], [Next], [Next], [Next], [Next], [Add User...], type "user", [OK], [Next], [Finish], [OK].
If your CD/DVD burner showed up in Writer category during setup, your setup was ok. Otherwise, something is screwy with the CD/DVD setup.

Click "Print system currently used:" drop-down menu > CUPS
In top menu, click Add > Add Printer/Class... > [Next] > "Local printer" > [Next] > highlight your printer, [Next] > highlight a manufacturer and a model, [Next] > [Settings...] > change your preferences, [OK], [Next] > [Next] > set up a quota if needed, [Next] > [Next] > type a printer name and location, [Next] > [Finish]
