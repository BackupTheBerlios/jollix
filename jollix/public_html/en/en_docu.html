0. Was ist jollix
Jollix ist eine freie GNU/Linux Distribution, die direkt von CD läuft (LiveCD).
Jollix ist ein speziell für Spiele und Multimedia vorkonfiguriertes Gentoo Linux,
das automatisch gängige PC-Hardware erkennen und konfigurieren kann. Die LiveCD
startet automatisch eine grafische Oberfläche (XFree mit KDE), so dass alle
wichtigen Anwendungen direkt benutzt werden können. Die Version 0.2 ist die zweite
Beta-Release von jollix. Die wichtigsten Erweiterungen gegenüber der Version 0.1.1
sind die Unterstützung von ATI Radeon Grafikkarten, ISDN-Karten und der FritzCard
DSL von AVM.

1. Netzwerk einrichten
Über "jollix einrichten" das Icon "Netzwerk einrichten" anklicken. "eth0" bezeichnet
die erste Netzwerkkarte im Rechner, "eth1" die zweite usw. Befindet sich nur eine
Netzwerkkarte im Rechner, kann die Vorgabe "eth0" übernommen werden.

2. ADSL einrichten
Die ADSL Verbindung wird eingerichtet über "ADSL einrichten". Es öffnet sich ein
Fenster, in dem das root-Passwort abgefragt wird. Danach wird man durch die ADSL
Einrichtung geführt. Mit "ADSL starten" wird anschließend die Verbindung aufgebaut.

T-Online DSL Kunden sollten den Benutzernamen wie folgt angeben:
1. Anschlusskennung
2. Zugehöriger T-Online Nummer
3. Suffix #0001
4. @t-online.de
z.B. 111111111111222222222222#0001@t-online.de

Befindet sich im Rechner mehr als eine Netzwerkkarte, gilt das gleiche
wie unter 2. "Netzwerk einrichten". Die erste Karte im Rechner ist "eth0",
die zweite "eth1" usw.

3. AVM Fritz DSL Karten
Neu in jollix 0.2 ist die Unterstützung der AVM FritzCard DSL. Die Einrichtung
einer DSL Verbindung mit einer AVM FritzCard DSL erfolgt über "jollix einrichten"
Fritz!CardDSL einrichten".

4. Half-Life und HL Modifikationen einrichten
Mit dem jollix "MODS Installer" kann eine Desktop-Verknüpfung mit der Half-Life
Installation auf der Festplatte eingerichtet werden. Dazu den Installationsordner
von Half-Life angeben, in dem sich hl.exe bzw. cstrike.exe befindet. Die Festplatten
werden in dem Dialog so oder ähnlich erscheinen: Partition1, Partition2 usw. Achtung:
Alle Partitionen sind schreibgeschützt und können von jollix nicht verändert werden!
STEAM wird noch nicht von jollix unterstützt. Spielbar sind nur die Half-Life und
HL-MODs Versionen aus der Vor-STEAM-Zeit!
jollix nutzt seit der Version 0.2 wine und nicht mehr winex! Damit wird es möglich,
alle HL MODS mit Cheathing Death zu spielen. Neu in jollix ist ab der Version 0.2
folgendes: Alle MODS laufen außerhalb des KDE Desktops in einem 2. Bereich.
D.h. man kann während des Spielens zurück in den Desktop gehen, chatten, im Web
browsen und sich nach Game-Servern umschauen. Das Umschalten zw. Spiel und
KDE Desktop funktioniert über die Tastenkombination Strg + Alt + F7 bzw. Strg + Alt + F8.
Besitzer einer NVidia Karte finden ihren KDE Desktop auf Strg + Alt + F7, das Spiel
auf Strg + Alt + F8. Besitzer einer ATI Karte finden den Desktop auf Strg + Alt + F8,
das Spiel auf Strg + Alt + F7 (Achtung: Boot Option "atigamer").

5. ATI Radeon Karten Unterstützung
ATI Kartenbesitzer können jollix 0.2 mit und ohne Bootparameter starten. Ohne
Bootparameter erscheint nach dem Booten der bekannte KDE Desktop. Ein Spielen
von MODS ist so jedoch nicht möglich, allerdings ist der KDE Desktop OpenGL und
Video xv beschleunigt. Zum Spielen von HL MODS mit ATI-Karten, muss jollix mit
dem Bootparameter "jollix atigamer" gebootet werden. Erklärt wird dies auch am
Bootprompt mit F2.

6. Passwörter
Die Passwörter für die beiden eingerichteten Benutzer sind vorgegeben. Der Benutzer
"root" ist vergleichbar mit dem "Administrator" unter Windows, daher sollte man
nur dann "root" benutzen, wenn es von jollix ausdrücklich so verlangt wird.
Der Benutzer "user" ist der eingerichtete Standardbenutzer, mit dem man spielen
und surfen sollte. Vor der Einrichtung eines Internetzugangs unter
jollix (Netzwerk oder ADSL), sollten die beiden Passwörter unbedingt geändert werden!
Nach einem Neustart der jollix LiveCD sind die beiden Passwörter allerdings wieder
zurückgesetzt, da sie nicht dauerhaft auf der CD gespeichert werden können.
Das root-Passwort und das user-Passwort ändert man am einfachsten, indem man
in "jollix einrichten" die "Benutzerverwaltung" aufruft und dort in der Liste den
Benutzer "user" bzw. "root" per Doppelklick auswählt.

Das voreingestellte root-Passwort lautet: jollix

Das voreingestellte user-Passwort lautet: user

7. Installation von jollix 0.2
Die jollix Version 0.2 dient auch zum Testen einer Installation von jollix auf der Festplatte.
Die ausführliche Anleitung gibt es hier:
http://www.jollix.de/de/de_docu.html#installation
Die Installation dient nur zu Testzwecken und kann nur von erfahrenen
Linuxanwendern durchgeführt werden! Ein grafischer Installer wird es erst in
einer der späteren jollix Versionen geben!

Hier klicken.

Was ist Gentoo?

Gentoo Linux ist eine vielseitige und schnelle, komplett freie Linux Distribution, die für Linux Power-User gedacht ist. Im Gegensatz zu anderen Distributionen hat Gentoo Linux ein fortschrittliches Paketverwaltungs-System namens Portage. Portage ist ein Ports-System in der Tradition von BSD-Ports. Es ist in Python realisiert und bringt einige zusätzliche Features mit, darunter auch ein fein granuliertes Management von Abhängigkeiten, die Möglichkeit, "fake"-Installationen durchzuführen, ein Pfad-Sandboxing, sicheres "Unmerging", System-Profile, virtuelle Pakete und ein Konfigurationsdatei-Management.
Gentoo Linux bildet die Basis von jollix und ist auf www.gentoo.org sehr gut dokumentiert.

Kann ich jollix gefahrlos nur mal eben ausprobieren?

Ja. Jollix kann "nur-lesend" direkt von CD-ROM gestartet werden. Dabei wird das bestehende System nicht verändert. Unter Windows installierte Programme können teilweise sogar gestartet werden, z.B. Counter-Strike.

Was ist wine?

Wine ist eine Implementierung der Windows Win32 und Win16 APIs auf Basis des Fenstermanagers X und Unix. Wine stellt unter Linux eine Umgebung zur Verfügung, in der Windows-Programme (.exe Dateien) auch unter Linux laufen. Wine benötigt keine Installation von Windows, da es eine hundertprozentige Unabhängigkeit vom Microsoft Code hat. Optional kann es die System DLL`s zur besseren Unterstützung der Windowsprogramme nutzen. Wine kommt mit dem Quellcode, mit Dokumentationen, Beispielen und es ist frei einsetzbar.
Nicht alle Windows-Programme, insbesondere neuere, sind jedoch auch garantiert lauffähig. So sind nicht alle Funktionen der neuesten Windows Versionen nachgebildet, die von einigen Programme zwingend benötigt werden. In jollix ist wine so eingerichtet, dass die Spiele und Programme, die in der jollix-wine-liste stehen, garantiert laufen. Eine deutschsprachige Seite über wine ist www.linux-wine.de, die Original Seiten sind auf www.winehq.com zu finden.

Was ist KDE?

KDE ist eine moderne grafische Arbeitsumgebung für UNIX Workstations. KDE strebt an, den Bedarf für eine einfach zu bedienende Arbeitsfläche für UNIX Workstations zu decken, ähnlich den Arbeitsumgebungen, die man unter MacOS oder Windows 9x/NT/2000/XP vorfindet. Alles Wissenswerte zu KDE gibt es auf www.kde.de nachzulesen.

Jollix Ausprobieren ohne Installation

Die LiveCD wird einfach ins CD-Rom oder DVD-Laufwerk des Rechners eingelegt. Im BIOS muss als erstes boot device "CDROM" o.ä. ausgewählt sein. Dann fährt jollix von der CD hoch und der Benutzer "user" wird automatisch angemeldet. Siehe auch die Readme.txt der aktuellen Version.

jollix Linux Installationsanleitung

Wichtige Anmerkungen:

Die Installation von jollix auf der Festplatte erfolgt auf eigene Gefahr! Die folgende Anleitung bezieht sich auf die jollix Version 0.2 und ist nur eine Anleitung, ein grafischer Installer wird es in einer der nächsten jollix Versionen geben. Die Anleitung kann auch zur testweisen Installation von jollix 0.1.1 benutzt werden, allerdings können zwei Konfigurationsschritte am Ende der Anleitung nicht mit dieser Version durchgeführt werden. Jollix 0.1.1 kann dann zwar uneingeschränkt von der Festplatte gebootet und benutzt, aber nicht aktualisiert werden!

Jollix kann mit dieser Anleitung nur auf einer völlig unbenutzten Festplatte installiert werden bzw. auf einer vorbereiteten Festplatte, die noch Platz für weitere Partitionen bietet. Befindet sich bereits ein anderes Betriebssystem auf der Festplatte und wird von diesem die gesammte Festplatte belegt, so bleibt nur eine Neupartitionierung oder eine Verkleinerung dieser Partition. Sowohl eine Neupartitionierung als auch eine Verkleinerung einer Partition sollte bedacht und vorsichtig durchgeführt werden! Eine Verkleinerung der bestehenden Partition ist nicht unter jollix 0.2 möglich! Die Installation parallel zu einem bestehenden System ist jedoch kein Problem, sofern es möglich ist, neue Partitionen anlegen zu können. In diesem Fall ist es sehr ratsam, ein komplettes Backup des bestehenden Systems zu machen und dieses separat, z.B. auf einer CD zu sichern bzw. alle wichtigen Daten des Systems extern zu sichern. Die Anleitung erläutert auch die Installation und Konfiguration eines Bootmanagers, der es ermöglicht beide Betriebsysteme aus einem Bootmenü heraus getrennt zu booten.

Das Team von jollix übernimmt keine Verantwortung für Schäden am Rechner und/oder den Verlust von Daten durch die Installation von jollix. Die Installation geschieht auf eigene Gefahr!

Alle nun folgendenen Installationsschritte müssen als Superuser (root) in einer Konsole des gebooteten jollix Systems durchgeführt werden. Hierzu öffnet man ein Konsolenfenster und meldet sich als Superuser mit dem Befehl su - an [1.1].

1.1: Als Superuser root in einem Konsolenfenster anmelden, Befehl: su -

 
user@jollix user # su -

Wie im nächsten Konsolenausschnitt zu sehen ist [1.2], sollte sich jetzt der Prompt geändert haben [jollix root # ], so dass man jetzt Superuser ist und mit der eigentlichen Installation beginnen kann. Zuerst müssen die beim Booten von jollix eingebundenen Partitionen unmountet werden, d.h. diese aus dem System abmelden, so dass kein Zugriff mehr besteht [1.2]:

jollix root # umount /mnt/transit/ide*

Anschliessend kann man prüfen, dass wirklich keine Festplatten Partitionen mehr eingebunden sind [1.3]:

1.3: Aktuell noch eingebundene Dateisysteme anzeigen, Befehl: mount

2.Festplatte partitionieren

ACHTUNG: Wie bereits in der Einleitung angekündigt, übernimmt das Team von jollix keine Verantwortung für Schäden und/oder Datenverlust durch das Partitionieren der Festplatte! Durch fehlerhafte Partitionierung der Festplatte können alle darauf enthaltenen Daten verloren gehen. Die Partitionierung geschieht vollkommen auf eigene Gefahr!

Im Folgenden soll kurz erläutert werden, was unter Partitionieren der Festplatte zu verstehen ist, in der für Linux typischen Benennung von Festplatten und Laufwerken und zwei mögliche Partitionierungsmöglichkeiten aufgezeigt werden.

Unter Partitionierung versteht man das Aufteilen der Festplatte in verschiedene Bereiche, die einzeln formatiert werden können. So entstehen durch Partitionierung einer 40GB Festplatte in 4 gleich grosse Teile 4 neue Festplatten zu 10GB. Die Partitionierung der Festplatte kann jederzeit wieder rückgängig gemacht werden. Befindet sich eine Festplatte im Rechner, so wird diese in Linux als /dev/hda angesprochen, sofern sie als Master am ersten IDE Kanal angeschlossen ist. Eine zweite am ersten IDE Kanal als Slave angeschlossene Platte würde demnach als /dev/hdb, die Master Platte am zweiten IDE Kanal als /dev/hdc und wiederum die Slave Platte als /dev/hdd bezeichnet werden. In den meisten Fällen werden am zweiten IDE Kanal CD/CD-RW/DVD -Laufwerke angeschlossen sein, diese werden dann ebenso bezeichnet mit hdc bzw. hdd. Die möglichen Partitionen einer Festplatte werden durchnummiert. Befinden sich auf hda insgesammt 3 Partitionen, so werden diese mit hda1, hda2 und hda3 bezeichnet. In den nun folgenden zwei Bespielen wird davon ausgegangen, dass sich erstens überhaupt keine Partition auf der Festplatte befindet, diese also unpartitioniert ist, bzw. dass die darauf vorhandenen Partitionen gelöscht werden können. Das zweite Beispiel geht davon aus, dass sich auf der Platte ein anderes Betriebssytem in der 1. Partition befindet, der Rest der Festplatte aber unpartitioniert ist. Für den Fall, dass eine einzige Partition, die gesamte Festplatte belegt, bleibt nur eine komplette Neupartitionierung. Wichtig: Dabei gehen alle Daten auf der Festplatte verloren!

Beim Partitionieren unterscheidet man primäre und erweiterte Partitionen. Maximal können 4 primäre Partitionen angelegt werden (hda1 bis hda4). Sollen/Müssen mehr als 4 Partitionen angelegt werden, muss spätestens die 4. Partition eine erweiterte Partition werden, in welcher dann wiederum logische Partitionen angelegt werden können.

1. Beispiel: Im Rechner befindet sich eine vollkommen unbenutzte Festplatte, die komplett für jollix genutzt werden kann und keine weiteren Partitionen enthält bzw. vorhandene gelöscht werden können. Eine minimale und sinnvolle Partitionierung besteht aus 3 Partitionen, nämlich aus einer boot- (hda1), einer swap- (hda2) und einer root-Partition (hda3):

Partition Grösse (empfohlen) Dateisystem (empfohlen) Device Bezeichnung
boot Partition mit Kernel und Bootkonfiguration 32 MB ext2 /dev/hda1
swap Partition 512 MB Linux swap /dev/hda2
root Partition mit dem jollix System >=2 GB ext3 /dev/hda3

2. Beispiel: In der 1. Partition der Festplatte befindet sich ein fremdes Betriebssystem, die Festplatte enthält jedoch noch freien Platz zum Anlegen der notwendigen jollix Partitionen. Die Nummerierung der Partitionen verschiebt sich daher um eins nach hinten, boot- (hda2), swap- (hda3) und root-Partition (hda4):

Partition Grösse (empfohlen) Dateisystem (empfohlen) Device Bezeichnung
Anderes Betriebssystem Zu viele GB FAT/NTFS /dev/hda1
boot Partition mit Kernel und Bootkonfiguration 32 MB ext2 /dev/hda2
swap Partition 512 MB Linux swap /dev/hda3
root Partition mit dem jollix System >=2 GB ext3 /dev/hda4

3.Dateisysteme anlegen

Die zuvor angelegten Partitionen werden nun formatiert, d.h. Dateisysteme darauf angelegt. Für die boot-Partition wird ext2 [3.1] und für die root-Partiton ext3 [3.3] empfohlen. Dies ist nur eine Empfehlung, es existieren weitere Dateisysteme, die aber hier jetzt nicht besprochen werden können. Nur soviel: Der später installierte jollix Kernel unterstützt alle zur Zeit bekannten Linux Dateisysteme (inkl. XFS, JFS usw.). Die swap-Partition wird unter [3.2] eingerichtet. Hier ist es sehr wichtig, die richtigen Device Bezeichnungen der eben angelegten Partitionen zu benutzen, sonst droht Datenverlust!

4.jollix auf der Festplatte installieren

Die Festplatten Partitionen sind jetzt für die Installation von jollix vorbereitet, so dass nun das System auf der Festplatte eingerichtet werden kann, vorher müssen jedoch noch die neuen Partitionen eingebunden werden, damit man sie auch ansprechen kann [4.1 bis 4.4]:

4.1: jollix Installationsverzeichnis anlegen, Befehl: mkdir

jollix mnt # mkdir jollix

4.2: Root Partition (hda3) einbinden, Befehl: mount

jollix mnt # mount /dev/hda3 jollix

4.3: Boot Verzeichnis (hda1) anlegen, Befehl: mkdir

jollix mnt # mkdir jollix/boot

4.4: Boot Partition (hda1) einbinden, Befehl: mount

jollix mnt # mount /dev/hda1 jollix/boot

/dev/hda3 on /mnt/jollix type ext3 (rw)
/dev/hda1 on /mnt/jollix/boot type ext2 (rw)

In den letzten beiden Zeilen der Ausgabe ist zu sehen, dass /dev/hda3 nach /mnt/jollix und /dev/hda1 nach /mnt/jollix/boot gemountet wurde. Sollte die eigene Ausgabe nicht mit der obigen übereinstimmen, wurde entweder eine andere Partitionierung gewählt oder es liegt ein Fehler vor.

Jollix kann nun von der CD auf die Festplatte kopiert werden [4.6 bis 4.7]:

4.6: In das jollix LiveCD Verzeichnis wechseln: cd

jollix mnt # cd cloop

4.7: jollix auf die Festplatte kopieren, Befehl: cp

jollix cloop # cp -dvR * /mnt/jollix/ --preserve=mode --preserve=ownership

Der nun folgende Kopiervorgang kann einige Minuten in Anspruch nehmen. Die über den Bildschirm laufenden Zeilen sind vom Kopierbefehl (Parameter v für verbose). Bitte darauf achten, dass der Befehl richtig eingegeben wurde.

5.Abschliessende Konfiguration der jollix Festplatteninstallation

Nachdem nun jollix auf die Festplatte kopiert wurde, wird ein chroot durchgeführt, so dass man direkt aus dem neuen jollix System heraus Änderungen und Konfigurationen durchführen kann. Die folgenden Befehle der Reihe nach durchführen [5.1 bis 5.6]:

5.1: In das Verzeichnis der neuen jollix Festplatteninstallation wechseln, Befehl: cd

jollix cloop # cd /mnt/jollix

5.2: proc Verzeichnis in die neue jollix Installation einbinden, Befehl: mount

jollix jollix # mount -t proc proc /mnt/jollix/proc

5.3: Kopieren der Konfigurationsdatei resolv.conf, Befehl: cp

jollix jollix # cp /etc/resolv.conf /mnt/jollix/etc/resolv.conf

5.4: In die neue jollix Festplatteninstallation wechseln, Befehl: chroot

jollix jollix # chroot /mnt/jollix /bin/bash

5.5: env-update

jollix / # env-update

Ausgabe von env-update:

Regenerating /etc/ld.so.cache...

5.6: Profil laden

jollix / # source /etc/profile

Nun befindet man sich direkt im neuen jollix System auf der Festplatte. Sollte in den nun folgenden Schritten ein Fehler auftreten oder zu einem späteren Zeitpunkt, ist es nicht notwendig alle bereits durchgeführen Schritte zu wiederholen, sondern man kann jederzeit wieder bei [5.1] in das neue Festplatten System von der LiveCD aus per chroot wechseln und so Fehler korrigieren. Allerdings müssen die Partitionen zuvor, wie unter [4.1] bis [4.4] erläutert, gemountet werden.

Im nächsten Schritt wird der Bootmanager grub installiert und konfiguriert. Dieser wird in den MBR (Masterbootrecord) der Festplatte installiert und bietet die Möglichkeit, auch andere Betriebssysteme zu booten. Das Installationsprogramm von grub wird direkt in der Konsole aufgerufen. Sobald es gestartet ist, werden 2 Befehle an grub geschickt. Es muss zunächst festgelegt werden, welche Partition die jollix boot-Partition ist, im ersten Beispiel oben wurde hda1 als boot-Partition vorgesehen. Daher lautet der Befehl root (hd0,0). Zur Erläuterung: hd0 bezeichnet die erste Festplatte am 1. IDE-Kanal, also hda. (hd0,0) steht folglich für die 1. Partition der 1. Festplatte. Grub fängt bei 0 mit dem Zählen an und nicht bei 1! Befindet sich in der 1. Partition ein anderes System, wie im 2. Beispiel erläutert wurde, und die jollix boot-Partition demnach in der zweiten Partition der Festplatte (hda2), ändert sich der Befehl zu root (hd0,1). Der zweite Befehl sagt grub, wohin grub installiert werden soll. Der Befehl lautet setup (hd0) und installiert grub in den MBR der 1. Festplatte. Die Konsolenausschnitte [5.7] und [5.8] zeigen die erläuterten Schritte für die Einrichtung von grub im MBR der 1. Festplatte:

5.7: Bootmanager grub starten, Befehl: grub

jollix / # grub

5.8: Bootmanager grub installieren


grub> root (hd0,0)
grub> setup (hd0)
grub> quit

Der Bootmanager grub ist nun installiert und muss im Folgenden nun noch für die verschiedenen Systeme, die wir booten wollen, konfiguriert werden. Dies geschieht in einer Textdatei, der grub.conf, die sich in im Verzeichnis /boot befindet. Die Einträge werden mit einem Texteditor gemacht. Empfohlen wird nano, der sehr leicht zu bedienen ist. Wichtig: Speichern und nano verlassen mit Strg+x.

5.9: Bootmanager grub konfigurieren, Befehl: nano

jollix / # nano -w /boot/grub/grub.conf

Die einfache grub.conf, wie unten abgebildet, kann sofort benutzt werden, sofern wie im 1. Beispiel nur jollix auf der Festplatte installiert wurde. Das zweite Beispiel bezieht sich auf eine Installation mit einem zweiten Betriebssystem in der 1. Partition der Festplatte. Die Nummerierung gleicht der oben beschriebenen, z.B. (hd0,1). Die grub.conf unten (einfache grub.conf) kann unverändert so übernommen werden, wenn die Partitionierung wie oben im 1. Beispiel gemacht wurde.

Einfache grub.conf (jollix 0.2):

default 0
timeout 30
splashimage=(hd0,0)/boot/grub/splash.xpm.gz

title=jollix starten
root (hd0,0)
kernel (hd0,0)/boot/vmlinuz-2.4.20-gaming-r3 root=/dev/hda3 splash=silent video=vesa:ywrap,mtrr vga=0x317
initrd=/boot/initrd-1024x768

Zweite, erweiterte grub.conf mit einem anderen System in der 1. Partition und der Einrichtung eines Bootmenüs zum Auswählen:

Erweiterte grub.conf mit Windows in der 1. Partition (hda1) und der jollix boot Partition in der 2. Partition (hda2):

default 0
timeout 30
splashimage=(hd0,0)/boot/grub/splash.xpm.gz

title=jollix starten
root (hd0,0)
kernel (hd0,0)/boot/vmlinuz-2.4.20-gaming-r3 root=/dev/hda3 splash=silent video=vesa:ywrap,mtrr vga=0x317
initrd=/boot/initrd-1024x768

title=Windows XP
root (hd0,0)
chainloader (hd0,0)+1

Im nächsten Schritt wird die Datei /etc/fstab entsprechend der oben durchgeführten Partitionierung konfiguriert. Wurde das 1. Beispiel mit drei Partitionen übernommen, fallen nur nur sehr wenig Änderungen an. Man öffnet /etc/fstab mit nano und entfernt die Kommentierungen (#) vor den /dev/hda Einträgen, so dass die Datei, wie unten gezeigt aussieht. Aus #/dev/hda1 wird /dev/hda1 ohne #. Wurde eine andere Partitionierung vorgenommen mit der jollix boot-Partition in der 2. Partition der Festplatte (hda2), so muss die Datei /etc/fstab entsprechend anpasst werden, boot (hda2), swap (hda3) und root (hda4).

5.10: fstab konfigurieren, Befehl: nano

jollix / # nano -w /etc/fstab

Einfache fstab (jollix 0.2):

/dev/hda1		/boot		ext2		noauto,noatime		1 1
/dev/hda3		/		ext3		noatime			0 0
/dev/hda2		none		swap		sw			0 0
/dev/cdroms/cdrom0	/mnt/cdrom0	iso9660		noauto,ro,user,unhide	0 0
/dev/cdroms/cdrom1	/mnt/cdrom1	iso9660		noauto,ro,user,unhide	0 0
/dev/fd0        	/mnt/floppy	vfat		noauto,ro,user,unhide	0 0
proc			/proc		proc		defaults		0 0
tmpfs			/dev/shm	tmpfs		defaults		0 0

jollix / # cp /spare/etc/local-org /etc/init.d/local

jollix / # cp /spare/etc/modules-org /etc/init.d/modules

Anschliessend den Rechner neu starten und die neue jollix Festplatteninstallation booten.

Dies sind: portage-spare.tar.bz2, kernel-spare.tar.bz2 und doc-spare.tar.bz2. Diese drei Dateien in /spare speichern und die nachfolgenden Befehle als root ausführen.
